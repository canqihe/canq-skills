# Bento Grid Design System

## 1. Design Philosophy

**"Grid-Based Modernity with Bold Impact — Where Information Meets Visual Drama"**

This is a contemporary dashboard-inspired design system that celebrates organized information hierarchy through grid layouts, oversized typography, and strategic color accents. The aesthetic draws from modern SaaS dashboards, Apple's design language, and the popular Bento Grid layout pattern.

### Core Aesthetic DNA

**Visual Language**: Grid-first layouts with cards containing content modules. High contrast between dark backgrounds and vibrant accent colors. Typography is the hero — massive numbers and headlines create visual impact and establish information hierarchy.

**Emotional Tone**: Professional, tech-forward, organized, and confident. The design feels like a premium dashboard or a modern product landing page — clean but never boring, structured but dynamic.

**Design Pillars**:
1. **Bento Grid Layout**: Modular card-based layouts where content is organized in a grid of bento box-like containers
2. **Oversized Typography**: Massive numbers (hero metrics) and headlines that demand attention and create hierarchy through scale alone
3. **Purple Accent System**: Deep purple (#673AB7) as the primary brand color, used strategically for emphasis and CTAs
4. **Linear Gradients**: Single-color gradients (accent fading to transparent) that add depth without mixing colors
5. **Clean Data Visualization**: Line-based charts and graphs that integrate seamlessly with the aesthetic
6. **Scroll-Triggered Motion**: Apple-inspired animations that reveal content as users scroll

### Interaction Philosophy

**Scroll Reveal**: Content should animate in as users scroll down — fade up, scale in, or slide into view. This creates a dynamic, engaging experience without overwhelming motion.

**Hover States**: Cards lift slightly, shadows deepen, and accent colors may intensify. Interactive elements respond but maintain the clean, professional aesthetic.

### The "Anti-Patterns" (What This Is NOT)
- **Not Cluttered**: Despite information density, the grid maintains breathing room
- **Not Rainbow**: Gradients are single-color only — no multi-color blends
- **Not Generic**: Typography scale and grid layouts create distinctiveness
- **Not Chaotic**: Everything aligns to the grid; nothing is randomly placed

## 2. Design Token System (The DNA)

### Colors (Dark Mode Primary)

**Philosophy**: Dark backgrounds create the canvas for content to pop. Purple is the hero accent, used strategically for emphasis.

*   **Background (Dark Base)**: `#0A0A0A` or `#111111` — Near-black for the main canvas
*   **Background (Card)**: `#1A1A1A` or `#1F1F1F` — Slightly lighter for card surfaces
*   **Foreground (Primary Text)**: `#FFFFFF` or `#FAFAFA` — High contrast for readability
*   **Foreground (Secondary Text)**: `#A0A0A0` or `#B0B0B0` — Muted for supporting text
*   **Primary Accent (Purple)**: `#673AB7` — The brand color. Used for:
  - Primary buttons
  - Highlights and emphasis
  - Chart lines
  - Icon accents
  - Active states
*   **Accent Gradient**: `linear-gradient(135deg, #673AB7 0%, rgba(103, 58, 183, 0) 100%)` — Single-color fade for backgrounds and overlays

**Neutral Colors**:
- Border: `#2A2A2A` or `#333333` — Subtle card borders
- Border (Hover): `#673AB7` — Purple borders on interactive states
- Divider: `#2A2A2A` — Section dividers

### Typography

**Font Philosophy**: Clean, modern sans-serif fonts with extensive weight variation. Typography scales dramatically — hero numbers can be 6xl-9xl while body text remains readable.

*   **Primary Font**: System font stack or Inter/Roboto
    ```css
    font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, "Helvetica Neue", Arial, sans-serif;
    ```
*   **Headings**: Weights 700-900 (Bold/Black)
*   **Body**: Weight 400-500 (Regular/Medium)
*   **Numbers/Metrics**: Weight 800-900 for oversized display

**Type Scale & Hierarchy**:
- **Hero Numbers/Metrics**: `text-6xl` to `text-9xl` (60px-128px) — These are the visual anchors
- **Page Headlines**: `text-4xl` to `text-6xl` (36px-60px) — Section titles
- **Card Titles**: `text-2xl` to `text-3xl` (24px-36px)
- **Body Text**: `text-base` to `text-lg` (16px-18px)
- **Small Text**: `text-sm` (14px) — Labels, metadata

**Text Colors**:
- Primary: `#FFFFFF` or `#FAFAFA`
- Secondary: `#A0A0A0` or `#B0B0B0`
- Muted: `#6B6B6B` or `#757575`
- Accent (links, highlights): `#673AB7`

### Radius & Borders

**Radius Philosophy**: Moderately rounded corners that feel modern but not playful.

*   **Border Radius**: `rounded-xl` (12px) or `rounded-2xl` (16px) — Cards and containers
*   **Button Radius**: `rounded-lg` (8px) or `rounded-xl` (12px)
*   **Border Width**: `border` (1px) for subtle dividers, `border-2` (2px) for emphasis
*   **Border Colors**:
  - Default: `border-[#2A2A2A]` or `border-[#333333]`
  - Hover: `border-[#673AB7]/50` or `border-[#673AB7]`

### Shadows & Effects

**Shadow Philosophy**: Subtle shadows that add depth without drama. Hover states lift cards with increased shadow.

*   **Card Shadow (Default)**: `shadow-lg` or custom `shadow-[0_4px_20px_rgba(0,0,0,0.3)]`
*   **Card Shadow (Hover)**: `shadow-xl` or `shadow-[0_8px_30px_rgba(103,58,183,0.15)]`
*   **Button Shadow**: `shadow-md` or `shadow-[0_2px_10px_rgba(103,58,183,0.3)]`
*   **Glow Effect**: Subtle purple glow on accent elements: `shadow-[0_0_20px_rgba(103,58,183,0.3)]`

### Background Effects

**Gradient Overlays**: Use single-color linear gradients that fade to transparent:
```css
background: linear-gradient(135deg, #673AB7 0%, rgba(103,58,183,0) 100%);
opacity: 0.1;
```

**Grid Patterns**: Optional subtle grid for tech feel:
```css
background-image: linear-gradient(rgba(255,255,255,0.03) 1px, transparent 1px),
                  linear-gradient(90deg, rgba(255,255,255,0.03) 1px, transparent 1px);
background-size: 50px 50px;
```

## 4. Bento Grid Layout Patterns

### Grid Structure

**Basic Bento Grid**:
```tsx
// 4-column grid with varying card sizes
grid grid-cols-1 md:grid-cols-2 lg:grid-cols-4 gap-4

// Card spanning
col-span-1  // 1 column
col-span-2  // 2 columns
row-span-2  // 2 rows tall
```

**Common Patterns**:
- **Hero Card**: Spans full width or 3 columns, contains headline + CTA
- **Metric Cards**: 2x1 or 1x1 cards with big numbers
- **Feature Grid**: Equal-sized cards in 2-4 columns
- **Asymmetric Layout**: Mix of 1x1, 2x1, 2x2 cards for visual interest

### Example Layout

```tsx
<div className="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-4 gap-4">
  {/* Hero card - 3 columns wide */}
  <div className="lg:col-span-3 bg-[#1A1A1A] rounded-2xl p-8">
    <h2 className="text-4xl font-black text-white">Headline</h2>
    <p className="text-[#A0A0A0] mt-4">Description</p>
  </div>

  {/* Metric card - 1 column */}
  <div className="bg-[#1A1A1A] rounded-2xl p-6">
    <div className="text-6xl font-black text-[#673AB7]">98%</div>
    <div className="text-sm text-[#A0A0A0] mt-2">Success Rate</div>
  </div>

  {/* Four 1x1 cards */}
  <div className="bg-[#1A1A1A] rounded-2xl p-6">Card 3</div>
  <div className="bg-[#1A1A1A] rounded-2xl p-6">Card 4</div>
  <div className="bg-[#1A1A1A] rounded-2xl p-6">Card 5</div>
  <div className="bg-[#1A1A1A] rounded-2xl p-6">Card 6</div>
</div>
```

## 5. Animation & Motion

**Philosophy**: Smooth, scroll-triggered animations that reveal content progressively. Inspired by Apple's product pages.

**Scroll Animations** (using Framer Motion):
```tsx
// Fade up on scroll
initial={{ opacity: 0, y: 30 }}
whileInView={{ opacity: 1, y: 0 }}
viewport={{ once: true, margin: "-100px" }}
transition={{ duration: 0.6, ease: "easeOut" }}

// Scale in
initial={{ opacity: 0, scale: 0.95 }}
whileInView={{ opacity: 1, scale: 1 }}
transition={{ duration: 0.5 }}

// Stagger children for grid
variants={{
  hidden: { opacity: 0 },
  visible: {
    opacity: 1,
    transition: {
      staggerChildren: 0.1
    }
  }
}}
```

**Hover Animations**:
```tsx
// Card lift
whileHover={{ y: -4, transition: { duration: 0.2 } }}

// Button scale
whileHover={{ scale: 1.02 }}
whileTap={{ scale: 0.98 }}
```

**Transition Timing**:
- Fade in: `duration: 0.5-0.6s`
- Hover: `duration: 0.2s`
- Ease: `easeOut` for natural feel

## 6. Data Visualization

**Chart Styling**:
- Line color: `#673AB7` (purple accent)
- Grid lines: Subtle `#2A2A2A` or `#333333`
- Text: `#A0A0A0` for labels, `#FFFFFF` for values
- Tooltips: `bg-[#1A1A1A] border border-[#2A2A2A] rounded-lg`

**Icon Style**:
- Simple line icons (outlined, not filled)
- Color: `#673AB7` for accent icons, `#A0A0A0` for neutral
- Size: `w-5 h-5` to `w-8 h-8` depending on context

## 7. Layout Strategy

**Container Width**: `max-w-7xl` (1280px) or `max-w-6xl` (1152px)

**Spacing System**:
- Section padding: `py-16` to `py-24` (64px-96px)
- Grid gaps: `gap-4` to `gap-6` (16px-24px)
- Card padding: `p-6` to `p-8` (24px-32px)
- Text margins: `space-y-4` to `space-y-6`

## 8. Responsive Strategy

**Breakpoints**: Mobile-first with `sm:`, `md:`, `lg:`

**Mobile Adaptations** (< 640px):
- Grids collapse to single column
- Hero numbers: `text-5xl` instead of `text-7xl`
- Cards: `p-4` instead of `p-6`
- Gaps: `gap-3` instead of `gap-4`

**Tablet** (640px - 1024px):
- Grids: 2 columns
- Medium typography sizes
- Maintain card structure

**Desktop** (> 1024px):
- Full grid layouts (3-4 columns)
- Maximum typography scale
- All hover effects active

**Key**: The bento grid should adapt gracefully — on mobile it's a stacked list, on desktop it's a true grid. Purple accents and bold typography remain consistent across all screen sizes.
